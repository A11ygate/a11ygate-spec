# A11yGate™ Category Positioning
_Version: 0.1 · Source: Brand Book v3 §1–§4_

## The Problem This File Solves

"Accessibility proof infrastructure" and "Verified Readability Infrastructure"
are both correct — but they are not at the same level. Mixing them creates
positioning confusion: one sounds like a standards body, the other sounds like
a typography tool.

## The Category Stack

```
Level 1 — Master category (always leads)
  "Accessibility proof infrastructure for governed digital systems"
  → Use in: hero, meta, all external-facing first impressions

Level 2 — Product category (inside the brand)
  "Accessibility standards body + verification layer + deployment gateway"
  → Use in: /standard, /proof, /about, investor materials

Level 3 — Module subcategory (never at hero level)
  "Verified Readability Infrastructure™" (typography module only)
  → Use in: /verified-readability, font module docs, Module 1 materials
```

## Application

| Context | Use |
|---|---|
| Site hero / meta description | Level 1 |
| Outreach posts, press | Level 1 → Level 2 if space |
| /standard page | Level 2 |
| /reference-font page | Level 2 → Level 3 |
| Font audit product pages | Level 3 |
| Investor one-pager | Level 1 + Level 2 |
| DEV.to article on TRS | Level 3 (technical audience) |

## The Typography Module Is Not A11yGate

The A11yGate Reference Font is Module 1 of a 13-module standard.
It is the most technically mature module at this stage.
That does not make it the product.
The product is accessibility proof infrastructure.
The font is a measurement instrument inside that infrastructure.
